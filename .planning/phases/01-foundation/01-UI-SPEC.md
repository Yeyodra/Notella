## Color
| Role | Light Mode Value | Dark Mode Value | Usage |
|------|------------------|-----------------|-------|
| Background (60%) | `0 0% 100%` (White) | `0 0% 3.9%` (Zinc 950) | Main app background, page surfaces |
| Surface/Card (30%) | `0 0% 96.1%` (Zinc 100) | `0 0% 14.9%` (Zinc 900) | Sidebar, cards, secondary containers |
| Primary/Accent (10%) | `0 0% 9%` (Zinc 900) | `0 0% 98%` (Zinc 50) | Primary actions, active states |
| Destructive | `0 84.2% 60.2%` (Red 500) | `0 62.8% 30.6%` (Red 900) | Delete buttons, error text |
| Muted Text | `0 0% 45.1%` (Zinc 500) | `0 0% 63.9%` (Zinc 400) | Secondary text, placeholders |
| Border | `0 0% 89.8%` (Zinc 200) | `0 0% 14.9%` (Zinc 800) | Dividers, card borders, input borders |

Accent reserved for: 
- Primary CTA buttons (Sign In, Sign Up, Create)
- Active sidebar links
- Active tabs
- Focus rings (`ring-primary`)
- Selected states (checkboxes, active toggles)

---

## Visual Hierarchy

- **Dashboard Shell**: The sidebar serves as the navigation anchor (secondary focal point), while the main content area acts as the primary focal point where user attention should be directed.
- **Auth Pages**: The authentication form is the primary focal point, centered on the screen to minimize distractions.
- **Icon-Only Actions**: All icon-only buttons MUST have an `aria-label` for accessibility. When the sidebar is in a collapsed state, it must show icons with tooltips to ensure actions remain clear.
